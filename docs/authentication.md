# Аутентификация

`yac` авторизуется OAuth-токеном Яндекса. В каждый запрос добавляется заголовок
`Authorization: OAuth <token>` (именно `OAuth`, не `Bearer`).

## Источники токена (по приоритету)

Токен ищется в трёх местах, первый найденный побеждает:

1. **Флаг `--token`** — `yac --token <OAUTH> segments list`
2. **Переменная окружения** `YANDEX_AUDIENCE_TOKEN`
3. **Файл** `~/.config/yandex-audience-cli/token` (одна строка, UTF-8)

```bash
# Вариант 1: флаг (разово)
yac --token y0_AgAA... segments list

# Вариант 2: переменная окружения (на сессию)
export YANDEX_AUDIENCE_TOKEN=y0_AgAA...
yac segments list

# Вариант 3: файл (постоянно)
mkdir -p ~/.config/yandex-audience-cli
echo 'y0_AgAA...' > ~/.config/yandex-audience-cli/token
yac segments list
```

Если токен не найден ни в одном источнике, команда завершится чистой ошибкой (без traceback)
с кодом возврата `1`.

## Где взять OAuth-токен

Создайте приложение на [oauth.yandex.ru](https://oauth.yandex.ru/) и выпустите токен **с правами
на Яндекс Аудитории**.

!!! warning "Один токен — но важны права (scopes)"
    OAuth-токен Яндекса **общий** для Директа, Метрики и Аудиторий — отдельные токены под каждый
    сервис не нужны. Доступ к конкретному API определяется **правами приложения**, под которым
    выпущен токен. Если `yac` отвечает `HTTP 403: Access is denied` при валидном токене — это не
    повод искать другой токен: у приложения просто нет права на Аудитории. Добавьте scope и
    перевыпустите токен.

## Файл `.env`

В репозитории есть `.env.example` — шаблон переменных. Скопируйте и заполните:

```bash
cp .env.example .env
# впишите YANDEX_AUDIENCE_TOKEN=...
```

!!! note "yac не читает `.env` автоматически"
    Зависимости `python-dotenv` нет — переменные нужно экспортировать вручную:
    ```bash
    set -a; . ./.env; set +a
    ```
    Этот способ корректно обрабатывает значения с пробелами и кавычками (в отличие от
    `export $(grep ... | xargs)`). Либо используйте [direnv](https://direnv.net/) с `.envrc: dotenv`.

    Реальный `.env` добавлен в `.gitignore` — токен не попадёт в репозиторий.

## Базовый URL

По умолчанию запросы идут на `https://api-audience.yandex.ru`. Переопределить (для тестов/прокси)
можно флагом `--base-url` или переменной `YANDEX_AUDIENCE_BASE_URL`. В обычной работе менять не нужно —
у API есть только версия `v1` и пространство `management`, это зафиксировано.
