# Разработка

## Тесты

```bash
pip install -e ".[dev]"
pytest                       # вся пирамида КРОМЕ e2e (e2e скипается без --run-e2e)
pytest --run-e2e             # + e2e (локальный stub-сервер; боевой — skip без YAC_E2E_TOKEN)
pytest tests/unit            # один уровень по пути
pytest -m smoke              # один уровень по маркеру
```

Тесты не требуют установки токена — HTTP мокается через [respx](https://lundberg.github.io/respx/).

## Тестовая пирамида

Тесты разложены по уровням `tests/{unit,integration,smoke,e2e}/`; уровень виден из `ls`, плюс на
каждом файле маркер `pytest.mark.<уровень>` (запуск и по пути, и по `-m`).

| Уровень | Что проверяет |
|---|---|
| **unit** | изолированные модули без сети: реестр (покрытие/дерево) и класс `Client` |
| **integration** | связку слоёв через respx-моки: resource↔client↔HTTP, multipart, команды CLI |
| **smoke** | дымовой прогон CLI без сети: справка, версия, структура групп, чистая ошибка без токена |
| **e2e** | сквозной прогон настоящего `yac` без моков (см. ниже) |

### e2e

e2e по умолчанию пропускается (помечается skip, пока не передан `--run-e2e`):

- `tests/e2e/test_local_server.py` — гоняет CLI против локального stdlib-сервера через `--base-url`
  (реальный сокет, без моков).
- `tests/e2e/test_live_api.py` — бьёт в боевой `api-audience.yandex.ru`; **пропускается** без
  переменной `YAC_E2E_TOKEN`.

```bash
pytest --run-e2e                         # локальный stub-сервер
YAC_E2E_TOKEN=... pytest tests/e2e/test_live_api.py --run-e2e   # боевой (read-only)
```

## Боевой smoke-прогон: `scripts/live_smoke.sh`

Bash-скрипт, прогоняющий **все 28 эндпоинтов** через настоящий `yac` на боевом Yandex Audience API.

!!! warning "Скрипт мутирует боевой аккаунт"
    `live_smoke.sh` создаёт и удаляет реальные объекты на вашем аккаунте. Безопасность обеспечена
    дизайном: скрипт **сам создаёт всё, что трогает** (объекты с префиксом `SMOKE yac`), трекает их
    и удаляет в `trap EXIT` — очистка срабатывает даже при падении. Существующие данные не
    затрагиваются. Боевой base-url зашит и пробрасывается явно; токен из `.env` не может уйти на
    чужой хост.

```bash
# токен в .env
bash scripts/live_smoke.sh
# ожидаемо: 30 PASS / 0 FAIL, в конце — cleanup всех созданных объектов
# (проверок больше, чем эндпоинтов: часть команд гоняется в нескольких сценариях — см. ниже)
```

Команды, которым нужен внешний ресурс (Метрика/AppMetrica/ClientId) или зрелый сегмент
(`reprocess`/`modify-data`), гоняются с заведомо невалидными/свежими данными и **ожидают ошибку
API** — это засчитывается как успех (проверяется, что CLI отправил запрос и разобрал ответ).

В CI скрипт не запускается (требует реального токена и квоты записи).

## Линтер

Форматирование — [black](https://black.readthedocs.io/):

```bash
black yac tests scripts
black --check yac tests scripts   # проверка без изменений
```
